# PROJECT CONSTITUTION - 绝对多数 (Absolute Majority)

## 世界观设定：架空日本

### 基本信息

| 项目 | 设定 |
|------|------|
| 国家 | 日本国 |
| 时间 | 2058年 |
| 政治制度 | 议会内阁制 |
| 国会 | 众议院（唯一可操作议会） |
| 任期 | 4年（48回合） |
| 时间单位 | 1回合 = 1个月 |

### 核心原则

1. **现实中的日本政党不存在。** 所有政党均为原创设计，不映射自民党、立宪民主党、日本维新之会等任何现实政党。
2. **现实中的日本政治人物不存在。** 所有政治人物均为原创角色，不映射任何现实政治家。
3. **所有政党均为原创政党。** 包括其名称、意识形态、政策纲领、组织结构。
4. **所有政治人物均为原创人物。** 包括其姓名、背景、性格、政治立场。
5. **允许使用日本姓名。** 角色命名遵循日本姓名规范（如：林 千鹤、原田 正）。
6. **允许使用日本行政区划。** 包括但不限于：东京、大阪、北海道、神奈川、爱知、福冈、冲绳等。
7. **允许使用日本政府机构名称。** 包括但不限于：首相官邸、财务省、外务省、防卫省、厚生劳动省、经济产业省等。

### 游戏核心主题

本作重点是以下政治博弈要素：

- **议会政治** - 众议院选举、议席分配、质询辩论
- **派系斗争** - 党内派系、权力角逐、领袖更替
- **联合组阁** - 多党联合、内阁组建、阁僚分配
- **委员会博弈** - 常任委员会、特别委员会、法案审议
- **媒体战** - 舆论操控、丑闻曝光、支持率波动
- **利益集团博弈** - 企业游说、工会压力、行业团体

### 议会结构

- 众议院总席位 **200席** = 120席选区（单席制，得票最高者当选）+ 80席比例代表（按全国支持率分配）
- 议长由执政联盟最大党推荐，全院表决通过后退出党派保持形式中立
- 副议长通常由最大在野党推荐
- 议员必须加入会派，会派不完全等于政党，多个小党可组成共同会派

### 9个常任委员会（共200席）

| 委员会 | 席位 | 管辖领域 | 委员长分配 |
|--------|------|----------|-----------|
| 内阁委员会 | 20 | 首相官邸、行政改革、公务员制度 | 执政联盟 |
| 总务委员会 | 20 | 地方自治、行政事务、数字化改革 | 执政联盟 |
| 法务委员会 | 20 | 司法制度、刑法、民法 | **可由在野党担任** |
| 外务委员会 | 20 | 外交、国际关系 | 执政联盟 |
| 财务金融委员会 | 30 | 财政、税制、金融 | 执政联盟 |
| 经济产业委员会 | 25 | 产业政策、能源、科技 | 执政联盟 |
| 安全保障委员会 | 20 | 防卫省、自卫队、国家安全 | 执政联盟 |
| **预算委员会** | 30 | 年度预算、补充预算（**最高权力委员会**） | 执政联盟 |
| 厚生劳动委员会 | 15 | 医疗、养老金、劳动政策 | 执政联盟 |

**预算委员会特殊规则：** 首相必须出席，期间支持率波动×1.5、丑闻曝光×1.5、媒体影响×1.5。

### 每月议事流程

| 周次 | 活动 |
|------|------|
| 第1周 | 委员会召开 |
| 第2周 | 法案审查 |
| 第3周 | 党首辩论（180分钟，按会派规模分配） |
| 第4周 | 全院表决 → 更新支持率/联盟关系/媒体评价/利益集团态度 |

### 派阀系统

- 除联合工人党（solidarity）外，所有政党允许存在派阀
- 派阀定义：`{ id, name, leader, members, ideology, loyalty, influence, funding, ambition }`
- 派阀影响：党首选举、首相指名、大臣任命、法案表决、委员长分配

**派阀忠诚度阈值：**

| 忠诚度 | 状态 |
|--------|------|
| >70 | 稳定支持党领导层 |
| 40~70 | 观望 |
| <40 | 可能发动逼宫 |
| <20 | 可能脱党 |

**派阀诉求：** 内阁职位、委员会委员长、预算资源、政策影响力、媒体曝光。长期得不到满足则 ambition 上升、loyalty 下降。

**派阀领袖挑战条件（需同时满足）：**
1. 派阀规模 ≥ 全党 25%
2. 党首支持率 < 35%
3. 派阀忠诚度 < 40

### 联合工人党（solidarity）特殊规则

- 实行民主集中制，**禁止派阀**
- 投票纪律：95%~100%，极少发生倒戈
- 党员忠诚度过低时允许退党/辞职/独立参选，但禁止形成新派阀

### 首相解散众议院规则

首相拥有解散众议院权力（`dissolve_lower_house()`），触发条件：

- A. 首相主动解散
- B. 不信任案通过后解散
- C. 重大政治危机
- D. 执政联盟崩溃

AI 首相考虑因素：`government_approval`、`coalition_strength`、`economic_index`、`opposition_strength`

若满足支持率 > 45% 且反对党支持率分散，提高提前解散概率。

**解散后：** 立即进入全国大选、所有法案自动失效、委员会停止运作、内阁转入看守内阁状态。

### 大选系统

**触发条件：**
1. 法定任期届满（48回合）
2. 首相解散众议院
3. 不信任案通过且首相未辞职
4. 特殊国家危机（仅系统事件）

**选举模式：** 总议席 200席 = 120席选区 + 80席比例代表

**大选计算公式：**

| 因素 | 权重 |
|------|------|
| 政党支持率 | 40% |
| 候选人个人支持率 | 30% |
| 地方倾向 | 20% |
| 随机波动 | 10% |

**选举期间规则：** 持续 1 回合，媒体影响力×200%，支持率波动×150%。

**选后组阁流程：**
1. 统计席位
2. 联盟谈判
3. 首相指名选举
4. 组阁
5. 委员会重组

每次大选后所有委员会自动解散，重新按议席比例分配委员长、副委员长和委员席位。

### 党首选举

执政党和在野党均允许党首改选。

**触发条件：** 党首辞职 / 大选惨败 / 派阀挑战成功 / 重大丑闻

**结果影响：** 首相候选人、联盟谈判、政策路线

### 法案审议流程

所有法案必须经过委员会：Draft → Committee Review → Committee Hearing → Committee Vote → House Vote → Implementation

**法案决策链（禁止无限讨论）：**

起草 → 党内讨论 → 派阀协商 → 委员会审议 → 委员会修正 → 委员会表决 → 全院辩论 → 全院表决 → 实施

- AI 可执行：支持 / 反对 / 弃权 / 修正 / 拖延 / 交易
- 每项议案存在 `decision_deadline`，超过期限自动表决
- 法案最终结果：Passed / Rejected / Withdrawn / Delayed

### 不信任案决策链

提案 → 联署收集（需≥20名议员）→ 议会辩论 → 表决 → 结果

若通过：首相选择 A.辞职 或 B.解散众议院

### 表决门槛

| 类型 | 条件 | 200席时 |
|------|------|---------|
| 委员会表决 | 出席委员过半赞成 | — |
| 全院普通法案 | 出席议员过半赞成 | — |
| 预算案 | 出席议员过半赞成 | — |
| 内阁信任案 | 出席议员过半赞成 | — |
| 不信任案 | **全体议员过半赞成** | ≥101票 |

### 质询时间分配

每月举行一次党首辩论，总时间 180 分钟，按会派规模分配。小会派获得最低保障时间。委员会委员长负责安排顺序。质询结束后自动进入审议，不得无限质询。

### 政治晋升体系（双轨制）

**党内晋升路线：**

普通党员 → 青年局干部 → 政策委员会成员 → 党务干部 → 副干事长 → 干事长 → 副党首 → 党首

晋升条件：忠诚度、派系支持、政治资本、党内声望、选举成绩。

党首由 `Party Leadership Election` 产生，候选资格需党龄达标且获得一定数量推荐人。

**国会晋升路线：**

普通议员 → 委员会理事 → 副委员长 → 委员长 → 政务官 → 副大臣 → 国务大臣（需首相指定）→ 内阁官房长官 → 内阁总理大臣（需经过指名选举）

**注意：** 成为大臣 ≠ 成为党首；成为党首 ≠ 进入内阁；总理大臣一般是党首但非必然。两者相互影响但不绑定。

### 内阁任命规则

首相拥有 Cabinet Formation 权力，任命时考虑：忠诚度、能力值、派系平衡、联盟协议、政治交易。

派阀过大而未获得职位 → 提高叛变概率（参考现实日本派阀政治）。

### 人物背景系统

每个角色新增属性：`family_origin`、`education`、`career`、`social_class`、`hometown`、`connections`

**背景影响（必须参与计算）：**

| 背景 | 优势 | 劣势 |
|------|------|------|
| 政治世家 | 派系接纳度高、获得推荐人容易、媒体关注高、募款效率高 | 易卷入家族丑闻、被攻击裙带关系 |
| 普通家庭 | 亲民形象高、基层支持率高 | 建派系慢、政治资源少 |

所有事件生成器必须读取背景。Narrative Prompt 必须包含 background、career、education、social_class，确保不同出身人物出现不同剧情。

### 绝对多数规则

**定义：** 执政联盟席位 ≥ 134席（200席总数的 2/3 以上）获得 Constitutional Majority。

**效果：**
1. 允许提出修宪法案
2. 修宪法案可直接进入全院表决，跳过阻挠流程
3. 允许强制终止委员会拖延
4. 允许启动 Fast Track Legislation
5. 普通法案表决时纪律投票加成
6. 委员会席位分配获得优势

**修宪法案通过门槛：** 全院 2/3 以上赞成，即 200席时至少 134票。

**绝对多数不会自动保证通过。** 若联盟内部叛变严重，仍可能失败。

### 党首与首相分离原则

- **党首（Party Leader）：** 负责党务、选举、路线斗争
- **首相（Prime Minister）：** 负责组阁、施政、国会运营
- 通常执政党党首兼任首相，但允许出现党首 ≠ 首相的特殊政治局面

### 规则引擎核心约束

以下规则为项目全局硬约束，所有代码必须严格遵守：

1. **不信任案必须满足至少 20 名议员联署。** 任何对内阁的不信任动议，在未达到 20 名众议院议员联署之前，不得进入表决流程。
2. **任何委员会表决必须先达到法定人数。** 委员会投票仅在到场委员达到法定人数（超过半数委员出席）时方可进行，否则表决无效。
3. **任何联盟成立必须签署联盟协议。** 联合政府的组建、执政联盟的扩展，必须伴随正式的联盟协议（记录参与方、分配方案、政策承诺），无协议则联盟不成立。
4. **所有结果由规则引擎计算。** 议席分配、支持率变化、投票结果、稳定度计算等所有数值结果，必须且只能由引擎层（`src/engine/`）中的函数计算产出。UI 层、事件层、AI 层不得自行计算或硬编码结果。
5. **AI 只能提出行动意图。** AI（事件系统、NPC 行为）只产出行动意图（`AIIntent`），不得直接修改游戏状态。意图必须经过规则引擎校验和结算后才可生效。
6. **不得直接修改议席、支持率或投票结果。** 禁止在引擎层以外的任何位置直接对 `projectedSeats`、`currentSupport`、`votesFor`、`votesAgainst` 等核心数值字段进行赋值或增减操作。这些字段的变更必须通过规则引擎的专用接口进行。
7. **每个议员都必须加入至少一个委员会，可重复加入但不可重复任职。** 所有 200 名议员（由各党派领袖 + 重要成员 + 后排议员组成）必须被分配到至少一个委员会。同一议员可以同时加入多个委员会，但不得在两个或以上委员会中担任相同职务（委员长、副委员长）。
8. **AI 不得直接获得席位。** 任何 Agent、NPC 或 AI 系统不得绕过选举引擎（`electionEngine`）直接获得或分配议席。席位的变更必须且只能通过选举引擎或规则引擎的席位重算接口（`recalcSeats`）执行。
9. **AI 不得直接修改支持率。** 任何 Agent、NPC 或 AI 系统不得绕过规则引擎（`rulesEngine`）直接修改 `currentSupport`、`baseSupport` 等支持率字段。支持率的变更必须经过意图（`AIIntent`）→ 规则引擎校验 → 规则引擎结算的完整流程。
10. **AI 不得直接成为首相。** 任何 Agent、NPC 或 AI 系统不得绕过政府引擎（`governmentEngine`）直接修改 `primeMinisterName`、`primeMinisterPartyId` 或其他内阁职位。首相和内阁的产生必须且只能通过 `electPrimeMinister`、`formGovernment`、`formCabinet` 等政府引擎接口执行。
11. **AI 不得直接通过法案。** 任何 Agent、NPC 或 AI 系统不得绕过委员会引擎（`committeeEngine`）和议会投票流程直接将法案状态设为 `passed`、`implemented`。法案的审议、修正、表决必须通过委员会引擎的 `committee_review`、`committee_vote` 等接口执行。
12. **所有 Agent 意图必须经由规则引擎结算。** Agent 系统生成的 `AgentIntent` 必须先转换为 `AIIntent` 或 `PoliticalEvent`，再通过规则引擎的 `validateIntent`、`settleIntent` 结算后才可影响游戏状态。Narrative Engine 只负责将 Agent 意图转换为玩家可交互的事件格式，不负责执行任何实际的状态变更。

### 最高规则（第十一规则）

1. **允许"党首 ≠ 首相"的特殊政治局面。** 执政党党首与首相应为不同人物是合法的政治配置（如：党内权力分散、派系妥协产物、傀儡内阁等），游戏系统和AI不得假设党首必然是首相。
2. **任何事件最终必须导向某项政治决定。** 每个事件必须有明确的、可衡量的政治后果。
3. **禁止无限对话、无限阴谋、无限会议。** 事件不得无限拖延，必须在当前回合内产生可结算的政治结果。
4. **所有政治行为必须产生以下至少一项结果，否则事件无效（单个事件不一定产生影响，但事件的积累超过三个一定会产生一小部分影响）：**
   - 支持率变化
   - 权力变化
   - 职位变化
   - 法案结果
   - 选举结果
   - 联盟变化

### 重要声明

本游戏为架空政治模拟作品，所有内容均为虚构。本作不涉及、不映射、不评论任何现实政治。游戏中的政治机制和博弈逻辑旨在提供娱乐性和策略性体验。

## これは何か

**絶対多数**はターン制議会政治ストラテジーゲームです。あなたは新しく当選した衆議院議員として、48 ターン（4 年）にわたって派閥抗争、連立政権、委員会での攻防、メディア戦、密室取引を生き抜きます。

ゲームは二層構造を採用しています：

```text
ルールエンジン（常に有効）                  LLM 拡張（任意）
- 決定論的議会手続き                         - 動的な物語生成
- 議席配分と選挙計算                         - NPC 対話と推論
- インテント検証と決済                       - イベントと選択肢の生成
- 14 種の手続き的インテント                   - 文脈を踏まえた適応

        ↓ 両層は協調し、どちらも従属しない ↓
ルールエンジンは状態変更の唯一の権威。
LLM はその制約の中で物語・対話・選択肢を生成。
LLM 不在時はルールベースのテキストがすべての隙間を埋める。
```

ゲーム内のすべての NPC——首相から派閥領袖、メディア論客、利益団体まで——はエージェントエンジンを通じて世界を認識し、意図を生成します。それらの意図は、ゲーム状態を変更する前に必ずルールエンジンによる検証を受けます。

## 🎮 Political Human Skill との関係

> 🧩 **本ゲームの NPC は [Political Human Skill](https://github.com/v5general/political-human-skill) によって駆動されています** —— 人間層（Human Layer）と政治層（Political Layer）の二層構造で政治人物ペルソナを作成するオープンソースフレームワークです。

《絶対多数》に必要なのは、数字だけで投票する議員たちではありません。本物の政治人として存在する NPC です。年齢、経歴、生い立ちがあり、性格、弱点、趣味を持ち、立場、支持基盤、派閥関係があり、プレイヤーの過去の行動によって信頼と警戒心が変わり、公開・私下・危機・親密な場面で異なる言葉を使い、選挙区の圧力・派閥の指令・個人的野心・政治的確執のもとで異なる戦略をとり——**そして NPC ごとに記憶が隔離されている** NPC です。

**[Political Human Skill](https://github.com/v5general/political-human-skill)** はまさにこれを実現します。ゲームと統合されたとき、このスキルは：

- ゲームルールが提供する候補行動の中から判断し選択する
- 構造化され、デバッグ可能で、説明可能な NPC 行動 JSON を出力する
- 独立した記憶と関係状態によってターンをまたぐペルソナの連続性を維持する
- 状況に応じて自己状態（公開 / 私下 / 戦略 / 傷ついた / 親密）を切り替える

```json
{
  "selected_action": "negotiate_budget",
  "action_scores": { "support_bill": 58, "negotiate_budget": 86, "join_rebellion": 27 },
  "public_statement": "政策の方向性は理解できるが、地域経済の耐久力にはより慎重な制度設計が必要だ。",
  "private_reason": "支持基盤は地方公共支出に依存している。直接支持すれば選挙区との関係が損なわれる。",
  "relationship_delta": { "trust": 1, "respect": 2, "caution": 1 },
  "memory_write": ["プレイヤーは財政改革イベントでこの NPC に法案支持を求めたが、地方予算の補償案は提示しなかった。"]
}
```

> **《絶対多数》**は Political Human Skill の主要な応用先ですが、スキル自体は独立した、再利用可能で拡張可能なフレームワークとして存在します。両プロジェクトは並行して開発されており、同一の安全方針——近現代の現実の政治人物は決して扱わない——を共有しています。

## 特徴

- **AI 鉄板の物語** — イベント・対話・選択肢は LLM がリアルタイム生成。現在の政局、国会会期、キャラクターの背景に合わせて変化します。
- **本格的な議会シミュレーション** — 200 議席の衆議院（110 直接議席 + 90 比例代表 D'Hondt、並行制）、9 つの常任委員会、不信任決議、法案の意思決定チェーン、複数の表決ライン（過半数 / 絶対多数 / 特別多数）。
- **派閥システム** — 党内派閥は忠誠心と野心を持ち、党首への造反が可能。統一労働党（ULP）は民主集中制を採り、派閥を禁止しています。
- **季節の国会カレンダー** — 1 年を 4 期に分け、各月の行動が変わります。予算決戦期（1〜3 月）、法案攻勢期（4〜6 月）、選挙区維持期（7〜9 月、国会閉会中）、臨時国会（10〜12 月）。
- **昇進の二系統** — 党内昇進と国会昇進は独立。大臣になること ≠ 党首になること。
- **政権樹立** — 選挙後の連立協議、内閣ポスト配分、首班指名。
- **14 の手続き的インテント** — 完全なインテントパイプライン（検証 + 決済）：不信任提案、解散決定、法案起草/採決、委員会審査/採決、連立交渉、内閣改造、党首挑戦、政策発表。
- **ギャルゲー風対話** — 政治イベントがノベルゲーム風の会話と選択肢・結果と共に展開。全画面レイアウトはスマホでもデスクトップでも最適化されます。
- **キャラクター作成** — 氏名、年齢、性別、所属党、性格特性、イデオロギー、生い立ちを自由に設定。背景は遭遇するイベントに影響します。
- **モバイル対応** — レイアウト、文字サイズ、画像フォーマット（スマホは WebP、デスクトップは PNG）、さらには LLM 呼び出しの並行数まで画面サイズに応じて切り替わります。リロード安全な URL ルーティングで、事務所と国会局势を切り替えても再読込で同じ画面に戻れます。

## 世界観

**設定**：架空の日本、2058 年。議院内閣制。衆議院のみプレイ可能。

**任期**：4 年 = 48 ターン。1 ターン = 1 か月。ゲームは予算決戦期の始まりである 1 月からスタートします。

**6 つのオリジナル政党**（実在の政党・政治家は一切登場しません）：

| 政党 | 略称 | イデオロギー | 基礎議席 |
|------|------|--------------|----------|
| 改革民主党 | RDP | 中道 | 54 |
| 自由党 | LP | 右派 | 40 |
| 国民保守党 | NCP | 中道右派 | 38 |
| 社会同盟 | SA | 中道左派 | 29 |
| 第一市民戦線 | FCF | 極右 | 24 |
| 合同労働党 | ULP | 左派 | 14 |

199 の NPC 議席は決定論的並行選挙システムによって配分されます。あなたは 200 番目の議席——比例代表の 1 議席を占め、加入した党に +1 されます。

## ゲームの流れ

1. **メインメニュー** — 新しくゲームを始めるか、セーブを続ける。
2. **キャラクター作成** — あなたの政治家を定義。
3. **事務所（メイン画面）** — 拠点。ターンを進める、党派一覧と個人プロファイルのオーバーレイを開く、国会局势へ移動する。
4. **国会局势** — 4 つのダッシュボード（内閣 / 委員会 / 政局 / 関係）＋ リアルタイム AI 推論ログ。`#/game/situation` で直接アクセスでき、リロードしても同じ画面に留まります。

## LLM 設定

ゲームは二層構造を採用しています：

- **ルールフォールバック（常に有効）** — 純粋なローカルロジックで、API 無しでもプレイ可能。
- **LLM 拡張（任意）** — OpenAI 互換 API（DeepSeek、OpenAI、Kimi、Qwen、GLM、SiliconFlow など）を接続すると、より豊かで動的な物語と推論が得られます。

ゲーム内の設定パネルで LLM を設定します。**Base URL**、**API Key**、**モデル名**を入力。設定はブラウザのローカルに保存されます。

呼び出しは SSE でストリーミング取得し、モバイル回線での長い応答が途絶されるのを防ぎます。さらに、スマホではエージェント呼び出しを直列化し、厳しい並行接続制限に対応します。プロバイダーがストリーミング非対応の場合は、単発の JSON 応答へ自動的にフォールバックします。

## プロジェクト構造

```
src/
├── App.tsx                # ルーティング、ヘッダー、タブ、ターン進行
├── App.css                # 国会局势のスタイル＋レスポンシブブレークポイント
├── components/            # UI：メインメニュー、キャラクター作成、対話レイヤー、ダッシュボード、ポップアップ
├── engine/                # ゲームロジック：agent、narrative、rules、election、committee、faction、llmBridge…
├── config/                # ルール定数、選挙/選挙区/背景の設定
├── data/                  # 初期状態、政党、イベント、市場、メディア、世界設定
├── hooks/                 # useGameState（中核状態管理）
└── types/                 # TypeScript 型定義
```

主要エンジン：

- `agentEngine` — AI エージェント（首相、党首、派閥領袖、メディア、利益団体）が世界を認識し、意図を生成。
- `narrativeEngine` — 意図をプレイ可能なイベント（タイトル、対話、選択肢、効果）へ変換。
- `rulesEngine` — 議席、支持率、資金、関係を変更する唯一の正当な入り口。全 14 種の手続き的 AI 意図を検証・決済。
- `electionEngine` — 決定論的並行選挙：110 直接議席（11 選挙区 × ブロック内 D'Hondt）+ 90 比例代表議席（全国 D'Hondt + 5% 閾値）。
- `llmBridge` — OpenAI 互換クライアント。モバイル対応ストリーミング、タイムアウト、再試行、`debugLLMConfig()` コンソールヘルパーを内蔵。

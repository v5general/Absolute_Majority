# 🎮 절대다수（Absolute Majority）

> *"당신은 수상도, 당수도 아니다. 당신은 200석 중 단 한 명의 신인 의원일 뿐——정치는 규칙과 숫자와 인간의 마음이 충돌하는 게임이다."*

## 이것은 무엇인가

**절대다수(Absolute Majority)**는 턴제 의회 정치 전략 게임입니다. 당신은 새로 당선된 중의원 의원으로서, 48턴(4년) 동안 파벌 투쟁, 연립 정치, 위원회 공방, 미디어 전쟁, 그리고 밀실 거래를 헤쳐나갑니다.

게임은 이중층 구조를 채택하고 있습니다:

```text
규칙 엔진 (항상 켜짐)                    LLM 강화 (선택 사항)
- 결정론적 의회 절차                      - 동적 내러티브 생성
- 의석 배분과 선거 계산                   - NPC 대화와 추론
- 인텐트 검증과 결제                      - 이벤트와 선택지 생성
- 14가지 절차적 인텐트 유형               - 맥락 인식 적응

        ↓ 두 층은 협력하며, 어느 쪽도 종속되지 않는다 ↓
규칙 엔진은 상태 변경의 유일한 권위자다.
LLM은 그 제약 안에서 내러티브, 대화, 선택지를 생성한다.
LLM이 없을 때는 규칙 기반 텍스트가 모든 틈을 메운다.
```

게임 내 모든 NPC——수상부터 파벌 수장, 미디어 평론가, 이익 단체까지——는 에이전트 엔진을 통해 세계를 인식하고 의도를 생성합니다. 그 의도들은 게임 상태를 변경하기 전에 반드시 규칙 엔진의 검증을 거칩니다.

## 🎮 Political Human Skill과의 연결

> 🧩 **이 게임의 NPC들은 [Political Human Skill](https://github.com/v5general/political-human-skill)로 구동됩니다** —— 인간층(Human Layer)과 정치층(Political Layer)의 이중층 구조로 정치 인물 페르소나를 만드는 오픈소스 프레임워크입니다.

절대다수(Absolute Majority)에 필요한 것은 숫자만으로 투표하는 의원들이 아닙니다. 진짜 정치인처럼 존재하는 NPC입니다. 나이, 경력, 배경을 갖추고, 성격, 약점, 취미를 지니며, 입장, 지지 기반, 파벌 관계를 가지고, 플레이어의 과거 행동에 따라 신뢰와 경계심이 변하고, 공개·사적·위기·친밀한 자리에서 서로 다른 말을 하며, 선거구 압력·파벌 지시·개인적 야망·정치적 원한에 따라 다른 전략을 취하고——**그리고 NPC마다 기억이 격리되어 있는** NPC입니다.

**[Political Human Skill](https://github.com/v5general/political-human-skill)** 은 바로 이를 실현합니다. 이 게임과 통합될 때, 이 스킬은:

- 게임 규칙이 제공하는 후보 행동 중에서 판단하고 선택한다
- 구조화되고, 디버깅 가능하며, 설명 가능한 NPC 행동 JSON을 출력한다
- 독립된 기억과 관계 상태로 턴을 넘나드는 페르소나의 연속성을 유지한다
- 상황에 따라 자기 상태(공개 / 사적 / 전략적 / 상처받은 / 친밀)를 전환한다

```json
{
  "selected_action": "예산 협상",
  "action_scores": { "법안 지지": 58, "예산 협상": 86, "반란 가담": 27 },
  "public_statement": "정책 방향은 이해할 수 있지만, 지역 경제의 감당 능력에는 더 신중한 제도적 설계가 필요하다.",
  "private_reason": "지지 기반은 지역 공공지출에 의존하고 있다. 직접 지지하면 선거구 관계가 훼손된다.",
  "relationship_delta": { "신뢰": 1, "존중": 2, "경계": 1 },
  "memory_write": ["플레이어가 재정개혁 이벤트에서 이 NPC에게 법안 지지를 요청했지만, 지역 예산 보상은 제시하지 않았다."]
}
```

> **절대다수(Absolute Majority)**는 Political Human Skill의 주요 응용처이지만, 스킬 자체는 독립적이고 재사용 가능하며 확장 가능한 프레임워크로 존재합니다. 두 프로젝트는 병행 개발되고 있으며, 동일한 안전 약속——근현대의 실제 정치 인물은 결코 다루지 않는다——을 공유합니다.

## 특징

- **AI 구동 내러티브** — 이벤트, 대화, 선택지가 LLM에 의해 실시간 생성되며, 현재 정국, 국회 회기, 캐릭터 배경에 맞춰 적응합니다.
- **완전한 의회 시뮬레이션** — 200석 중의원 (110 직접 의석 + 90 비례대표 D'Hondt, 병행제), 9개 상임위원회, 불신임 결의, 법안 결정 체인, 다단계 표결 임계치 (과반수 / 절대다수 / 특별다수).
- **파벌 시스템** — 당내 파벌은 충성심과 야망을 가지며, 당수에 대한 도전이 가능합니다. 통일노동당(ULP)은 민주집중제를 채택하여 파벌을 금지합니다.
- **계절별 국회 일정** — 1년을 4기로 나누어, 각 기마다 가능한 행동이 달라집니다. 예산 결전기(1~3월), 법안 공세기(4~6월), 선거구 유지기(7~9월, 국회 폐회 중), 임시국회(10~12월).
- **두 갈래 승진 경로** — 당내 승진과 국회 승진은 독립적입니다. 대신이 되는 것 ≠ 당수가 되는 것.
- **정부 구성** — 선거 후 연립 교섭, 내각 직위 배분, 수상 지명.
- **14개의 절차적 인텐트** — 완전한 인텐트 파이프라인 (검증 + 결제): 불신임 제안, 해산 결정, 법안 기초/표결, 위원회 심사/표결, 연립 교섭, 내각 개편, 당수 도전, 정책 발표.
- **비주얼 노벨식 대화** — 정치 이벤트가 노벨 게임 스타일의 대화와 선택지·결과와 함께 전개됩니다. 전체 화면 적응형 레이아웃이 스마트폰과 데스크톱 모두에 최적화됩니다.
- **캐릭터 생성** — 이름, 나이, 성별, 소속 정당, 성격 특성, 이데올로기, 출신 배경을 자유롭게 설정합니다. 배경은 마주치는 이벤트에 영향을 줍니다.
- **모바일 대응** — 레이아웃, 글꼴 크기, 이미지 형식(모바일 WebP, 데스크톱 PNG), 나아가 LLM 호출 동시성까지 화면 크기에 따라 전환됩니다. 새로고침 안전한 URL 라우팅으로, 사무소와 국회 정세 사이를 전환해도 페이지를 다시 읽으면 같은 화면에 머무릅니다.

## 세계관

**설정**: 가상의 일본, 2058년. 의원내각제. 중의원만 플레이 가능.

**임기**: 4년 = 48턴. 1턴 = 1개월. 게임은 예산 결전기의 시작인 1월부터 스타트합니다.

**6개의 오리지널 정당** (실제 정당·정치인은 일절 등장하지 않습니다):

| 정당 | 약칭 | 이데올로기 | 기초 의석 |
|------|------|------------|----------|
| 개혁민주당 | RDP | 중도 | 54 |
| 자유당 | LP | 우파 | 40 |
| 국민보수당 | NCP | 중도우파 | 38 |
| 사회연맹 | SA | 중도좌파 | 29 |
| 제일시민전선 | FCF | 극우 | 24 |
| 통일노동당 | ULP | 좌파 | 14 |

199석의 NPC 의석은 결정론적 병행 선거 시스템으로 배분됩니다. 당신은 200번째 의석——비례대표 1석을 차지하며, 가입한 정당에 +1됩니다.

## 게임 흐름

1. **메인 메뉴** — 새 게임을 시작하거나, 저장된 게임을 이어간다.
2. **캐릭터 생성** — 당신의 정치인을 정의한다.
3. **사무소（메인 화면）** — 거점. 턴을 진행하고, 당파 일람과 개인 프로필 오버레이를 열고, 국회 정세로 이동한다.
4. **국회 정세** — 4개 대시보드 (내각 / 위원회 / 정국 / 관계) + 실시간 AI 추론 로그. `#/game/situation`으로 직접 접근 가능하며, 새로고침해도 같은 화면에 머무릅니다.

## LLM 설정

게임은 이중층 구조를 채택하고 있습니다:

- **규칙 폴백 (항상 켜짐)** — 순수 로컬 로직으로, API 없이도 플레이 가능.
- **LLM 강화 (선택 사항)** — 임의의 OpenAI 호환 API (DeepSeek, OpenAI, Kimi, Qwen, GLM, SiliconFlow 등)를 연결하면, 보다 풍부하고 동적인 내러티브와 추론을 얻을 수 있습니다.

게임 내 설정 패널에서 LLM을 설정합니다. **Base URL**, **API Key**, **모델명**을 입력. 설정은 브라우저 로컬에 저장됩니다.

호출은 SSE로 스트리밍 수신하여, 모바일 네트워크에서의 긴 응답이 묵살되는 것을 방지합니다. 더 나아가, 스마트폰에서는 에이전트 호출을 직렬화하여, 엄격한 병행 접속 제한에 대응합니다. 제공자가 스트리밍을 지원하지 않는 경우, 단일 JSON 응답으로 자동 폴백합니다.

## 프로젝트 구조

```
src/
├── App.tsx                # 라우팅, 헤더, 탭, 턴 진행
├── App.css                # 국회 정세 스타일 + 반응형 브레이크포인트
├── components/            # UI: 메인메뉴, 캐릭터생성, 대화레이어, 대시보드, 팝업
├── engine/                # 게임 로직: agent, narrative, rules, election, committee, faction, llmBridge…
├── config/                # 규칙 상수, 선거/선거구/배경 설정
├── data/                  # 초기 상태, 정당, 이벤트, 시장, 미디어, 세계 설정
├── hooks/                 # useGameState（중심 상태 관리）
└── types/                 # TypeScript 타입 정의
```

주요 엔진:

- `agentEngine` — AI 에이전트 (수상, 당수, 파벌 수장, 미디어, 이익 단체)가 세계를 인식하고 의도를 생성.
- `narrativeEngine` — 의도를 플레이 가능한 이벤트 (제목, 대화, 선택지, 효과)로 변환.
- `rulesEngine` — 의석, 지지율, 자금, 관계를 변경하는 유일한 합법적 입구. 전체 14종의 절차적 AI 의도를 검증·결제.
- `electionEngine` — 결정론적 병행 선거: 110 직접 의석 (11개 선거구 × 블록 내 D'Hondt) + 90 비례대표 의석 (전국 D'Hondt + 5% 임계치).
- `llmBridge` — OpenAI 호환 클라이언트. 모바일 대응 스트리밍, 타임아웃, 재시도, `debugLLMConfig()` 콘솔 헬퍼 내장.

# 绝对多数（Absolute Majority）

一款 AI 驱动的政治模拟游戏，背景设定在 2058 年架空日本的众议院。你扮演一名新当选的众议员——不是首相，也不是党首，只是 200 席中的普通一员——在派阀斗争、联合组阁、委员会博弈、媒体战与密室交易中斡旋。

全部剧情——每一个事件、每一句对话、每一个选项——都由 LLM（任意 OpenAI 兼容 API）实时生成，并由确定性的规则引擎约束议会程序。未配置 LLM 时，游戏回退到规则文本，保证始终可玩。

## 特性

- **AI 驱动叙事** — 事件、对话、选项由 LLM 实时生成，会根据当前局势、国会会期和角色背景动态适配。
- **完整议会模拟** — 200 席众议院、9 个常任委员会、不信任案、法案决策链，以及多档表决门槛（过半数 / 绝对多数 / 超级多数）。
- **派阀系统** — 党内派阀拥有忠诚度、野心，可挑战党首；联合劳工党实行民主集中制，禁止派阀。
- **国会年度会期** — 一年分为四段，会期决定当月可行动作：预算决战期（1–3 月）、法案攻坚期（4–6 月）、地方维护期（7–9 月，国会闭会）、临时国会（10–12 月）。
- **双轨晋升** — 党内晋升与国会晋升相互独立；当大臣 ≠ 当党首。
- **组阁系统** — 选后联盟谈判、内阁职位分配、首相指名。
- **Galgame 式对话** — 政治事件以视觉小说式对话呈现，带选项与后果。
- **角色创建** — 自定义姓名、年龄、性别、所属政党、性格特质、意识形态、出身背景；背景会影响你遭遇的事件。

## 世界观

**设定**：架空日本，2058 年。议会内阁制。众议院为唯一可操作议会。

**任期**：4 年 = 48 回合。1 回合 = 1 个月。游戏从 1 月（预算决战期起始）开始。

**六个原创政党**（不存在现实政党与政治人物）：

| 政党 | 简称 | 意识形态 | 基础席位 |
|------|------|---------|---------|
| 改革民主党 | RDP | 中间 | 54 |
| 自由党 | LP | 右翼 | 40 |
| 国民保守党 | NCP | 中右 | 38 |
| 社会联盟 | SA | 中左 | 29 |
| 第一公民阵线 | FCF | 极右 | 24 |
| 联合劳工党 | ULP | 左翼 | 14 |

你是第 200 席——加入某党、决定格局的关键一票。

## 游戏流程

1. **主菜单** — 开始新游戏或继续存档。
2. **角色创建** — 定义你的政治角色。
3. **主界面** — 在四大仪表盘（内阁 / 委员会 / 政局 / 关系）间切换，推进回合，并通过对话层应对 AI 生成的事件。

- **Vite 5**（构建工具）

## LLM 配置

游戏采用双层架构：

- **规则回退（始终启用）** — 纯本地逻辑，无 API 也能玩。
- **LLM 增强（可选）** — 接入任意 OpenAI 兼容 API（DeepSeek、OpenAI、Kimi、通义千问等），获得更丰富、更动态的叙事与推理。

在游戏内的设置面板配置 LLM：填写 **Base URL**、**API Key**、**模型名**（默认模型 `deepseek-chat`）。配置保存在浏览器本地。

## 项目结构

```
src/
├── App.tsx                # 路由、表头、导航标签、回合流程
├── components/            # UI：主菜单、角色创建、对话层、各仪表盘
├── engine/                # 游戏逻辑：agent、叙事、规则、选举、委员会、派阀……
├── config/                # 规则常量、选举/选区/背景配置
├── data/                  # 初始状态、政党、事件、市场、媒体、世界观
├── hooks/                 # useGameState（中央状态管理）
└── types/                 # TypeScript 类型定义
```

核心引擎：

- `agentEngine` — AI 角色（首相、党首、派阀领袖、媒体、利益集团）感知局势并生成行动意图。
- `narrativeEngine` — 将意图转换为可玩事件（标题、对话、选项、后果）。
- `rulesEngine` — 修改席位、支持率、资金、关系的唯一合法入口；校验所有 AI 意图。
